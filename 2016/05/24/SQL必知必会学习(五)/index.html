12.联结表(SQL中最重要、最强大的特性，这块也有很多地方写作连接)

SQL最强大的功能之一就是能在数据查询的执行中联结表。

首先我们先要知道关系表的设计就是要把信息分解成多个表，一类数据一个表。各表通过某些共同的值相互关联（所以才叫关系数据库）。因此也可以知道关系数据库的可伸缩性远比非关系数据库要好。那么，如果数据存储在多个表中，怎样用一条SELECT语句就检索出数据呢？答案是用联结。

比如：

SELECT vend_name, prod_name, prod_price FROM Vendors, Products WHERE Vendors.vend_id = Products.vend_id;

这样两个表就联结起来了。当然，根据这个例子可能会觉得WHERE子句建立联结关系似乎有点奇怪，但是实际上是有个很充分的理由的。要记住，在一条SELECT语句中联结几个表时，相应的关系是在运行中构造的。如果没有WHERE子句，第一个表中的每一行将和第二个表中的每一行配对，而不管他们逻辑上是否能配在一起。所以，WHERE子句在建立联结时是很重要的。

上面我们所看到的联结称为等值联结，同时也称为内联结。当然，这种这种联结可以使用稍微不同的语法来明确的指定联结的类型。比如上一个语句可以写成如下：

SELECT vend_name, prod_name, prod_price FROM Vendors INNER JOIN Products ON Vendors.vend_id = Products.vend_id;

这两个语句是相同的。

SQL是不限制一条SELECT语句中所联结表的数目，例如：

SELECT prod_name, vend_name, prod_price, quantity FROM OrderItems, Products, Vendors WHERE Products.vend_id = Vendors_vend_id AND OrderItem.prod_id = Products.prod_id AND order_num = 20007;

虽然SQL本身不限制每个联结约束中表的数目，但是许多的DBMS都有限制。

其实上一章中，我们有一个例子：

SELECT cust_name, cust_contanct FROM Customers WHERE cust_id IN (SELECT cust_id FROM Orders WHERE order_num IN (SELECT order_num FROM OrderItems WHERE prod_id = 'RGAN01'));

可以用联结来做相同的查询：

SELECT cust_name, cust_contact FROM Customers, Orders, OrderItems WHERE Customers.cust_id = Orders.cust_id AND OrderItems.order_num = Orders.order_num AND prod_id = 'RGAN01';

13.创建高级联结

这一章主要介绍另外一些联结以及如何使用表别名，还有如何对被连接的表使用聚集函数。

在之前的学习中，知道了如何使用别名引用被检索的表列，如下：

SELECT vend_name + '(' + vend_country + ')' FROM Vendors ORDER BY vend_name;

SQL除了可以对列名和计算字段使用别名，还允许给表名起别名。这么做有以下两个理由：

  • 缩短SQL语句;
  • 允许在一条SELECT语句中多次使用相同的表。

使用方式如下：

SELECT cust_name, cust_contact FROM Customers AS C, Orders AS O ,OrderItems AS OI WHERE C.cust_id = O.cust_id AND OI.order_num = O.order_num AND prod_id = 'RGAN01';
使用不同类型的联结

自联结、自然联结、外联结

自联结：书中这块没有定义，但是按照笔者的理解就是将一张表看成多张表来做联结。

举个例子：

SELECT cust_id, cust_name, cust_contact FROM Customers WHERE cust_name = (SELECT cust_name FROM Customers WHERE cust_contact = 'Jim Jones');

这是利用了子查询的写法，下面我们来看一下用自连接应该怎么写：

SELECT c1.cust_id,c1.cust_name,c1.cust_contact FROM Customers AS C1, Customers AS c2 WHERE c1.cust_name = c2.cust_name AND c2.cust_contact = 'Jim Jones';

虽然这两种写法所得到的结果都一样，同样第一种写法也是合法的，但是对Customers的引用有了歧义性，因此采取表别名后，用连接比较合适。

自然联结

无论何时对表进行联结，应该至少有一列不止出现在一个表中（被联结的列）。标准的联结返回所有数据，相同的列甚至多次出现。自然联结排除多次出现，使每一列只返回一次。自然联结要求你只能选择那些唯一的列，一般通过对一个表使用通配符（SELECT*），而对其他表的列使用明确的子集来完成。继续举例：

SELECT C.*, O.order_num, O.order_date, OI.prod_id, OI.quantity, OI.item_price FROM Customers AS C, Order AS O, OrderItems AS OI WHERE C.cust_id = o.cust_id AND IO.order_num = O.order_num AND prod_id = 'RGAN01';

在这个例子中，通配符只对第一个表使用。所有其他列明确列出，所以没有重复的列被检索出来。

外联结

需要包含没有关联行的那些行就要用到外联结了。

下面举个例子，是一个简单的内连接：

SELECT Customers.cust_id,Orders.order_num FROM Customer INNER JOIN Orders ON Customers.cust_id = Orders.cust_id;

外联结语法类似：

SELECT Customers.cust_id, Orders.order_num FROM Customers LEFT OUTER JOIN Orders ON Customers.cust_id = Order.cust_id;

类似于内连接，这条SELECT语句使用了关键字OUTER JOIN来指定连接类型（而不是在WHERE子句中指定）。

使用带聚集函数的联结

直接上例子：要检索所有顾客及每个顾客所下的订单数。

SELECT Customers.cust_id, COUNT(Orders.order_num) AS num_ord FROM Customers INNER JOIN Orders ON Customers.cust_id = Orders.cust_id GROUP BY Customers.cust_id;

这条SELECT语句使用INNER JOIN将Customers和Orders表互相关联。GROUP BY子句按顾客分组数据，因此，函数调用COUNT(Orders.order_num)对每个顾客的订单技术，将它作为num_ord返回。 LEFT OUTER JOIN类似，可以将没有关联的行业联结返回。

最后总结一下使用联结和联结条件。

  • 注意所使用的联结类型。一般我们使用内连接，但是用外联结也有效。
  • 关于确切的联结语法，应该查看具体的文档，看相应的DBMS支持何种语法（大多数DBMS使用本文所描述的某种语法）。
  • 保证使用正确的连接条件（不管采用那种语法），否则会返回不正确的数据。
  • 应该总是提供联结条件，否则会得出笛卡尔积。
  • 在一个联结中可以包含多个表，甚至可以对每个联结采用不同的联结类型。虽然这样做是合法的，一般也很有用，但应该在一起测试它们前分别测试每个连接。这会使故障排除更为简单。

14.组合查询

多数SQL查询只包含从一个或多个表中返回数据的单挑SELECT语句。但是，SQL也允许执行多个查询（多条SELECT语句），并将结果作为一个查询结果集返回。这些组合查询通常称为并或复合查询。一般有两种情况需要使用组合查询：

  • 在一个查询中从不同的表返回结构数据。
  • 对一个表执行多个查询，按一个查询返回数据。

多数情况下，组合相同表的两个查询所完成的工作与具有多个WHERE子句条件的一个查询所完成的工作相同。换句话说，任何具有多个WHERE子句的SELECT语句都可以作为一个组合查询。

下面给出例子，先来两个单条语句：

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_state IN ('IL','IN','MI');

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_name = 'Fun4All';

然后可以使用UNION来进行这两条语句组合:

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_state IN ('IL','IN','MI') UNION SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_name = 'Fun4All';

再可以看一个使用WHERE子句查询的例子：

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_state IN ('IL','IN','MI') OR cust_name = 'Fun4All';

UNION在使用上很简单，只要注意以下几点即可：

  • UNION必须有两条或两条以上的SELECT语句组成，语句之间用关键字UNION分隔。
  • UNION中的每个查询必须包含相同的列、表达式或聚集函数。
  • 列数据雷星必须兼容：类型不必完全相同，但必须是DBMS可以隐含转换的类型。

包含或取消重复的行
UNION从查询结果中自动去除了重复的行，这个是UNION的默认行为。如果想返回所有匹配的行，那么可以使用 UNION ALL。例如：

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_state IN ('IL','IN','MI') UNION ALL SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_name = 'Fun4All';

提示：UNION总是完成与多个WHERE条件相同的工作。UNION ALL 为UNION的一种形式，他完成WHERE子句完成不了的工作。如果要返回所有匹配条件的行包括重复行，必须使用UNION ALL，而不是WHERE。

对组合查询结果排序

SELECT语句的输出用ORDER BY子句排序。在用UNION组合查询时，只能使用一条ORDER BY子句，他必须位于最后一条SELECT语句之后。对于结果集，不存在用一种方式排序一部分，而又用另一种方式排序另一部分的情况，因此不允许使用多条ORDER BY子句。使用方式可以如下:

SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_state IN ('IL','IN','MI') UNION SELECT cust_name, cust_contact, cust_email FROM Customers WHERE cust_name = 'Fun4All' ORDER BY cust_name, cust_contact;
