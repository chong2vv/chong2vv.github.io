15.插入数据

从之前写的笔记不难看出SELECT是常用的语句，但是数据库毕竟是增删改查。因此现在开始来学习增——数据插入（INSERT）。

INSERT用来将行插入（或添加）到数据库表中。插入有以下几种方式：

  • 插入完整的行
  • 插入行的一部分
  • 插入某些查询的结果

把数据插入表中的最简单方法是使用基本的INSERT语法，他要求指定表名和插入到新行中的值。例如：

INSERT INTO Customers VALUES('1000000006','Toy Land','123 Any Street','New York','NY','11111','USA',NULL,NULL);

各列必须以它们在表定义中出现的次序填充。所以这条语法并不安全，因此现在来介绍更安全的一种写法：

INSERT INTO Customers(cust_id, cust_name, cust_address, cust_city, cust_state, cust_zip, cust_country, cust_contact, cust_email) VALUES('1000000006','Toy Land','123 Any Street','New York','NY','11111','USA',NULL,NULL);

这两条语句的工作完全相同，但是表名后的括号里明确的给出了列名。在插入行时，DBMS将用VALUES列表中的相应值填入列表中的对应项。VALUES中的第一个值对应于第一个指定列名，第二个值对应于第二个列名，以此类推。他的优点是，即使表的结构发生改变，这条INSERT语句依然可以正常工作。

注意：无论使用哪种INSERT语句，VALUES的数目都必须正确。如果不提供列名，则必须给每个表列提供一个值；如果提供列名，则必须给列出的每个列一个值。否则，就会产生一条错误信息，相应的行不能成功插入。

同时，使用推荐的这种语法（第二种，明确给出表的列名），这种语法还可以省略列，直插入部分列，例如：

INSERT INTO Customers(cust_id, cust_name, cust_address, cust_city, cust_state, cust_zip, cust_country) VALUES('1000000006','Toy Land','123 Any Street','New York','NY','11111','USA');

但是，如果在INSERT操作中省略某些列需要表的定义允许，同时，省略的列必须满足以下某个条件：

  • 该列定义为允许NULL值（无值或空值）
  • 在表定义中给出默认值。这表示如果不给出值，将使用默认值。

小提示：如果表中不允许有NULL值或者默认值，这时却省略了表中的值，DBMS就会产生错误消息，相应的行不能成功插入。

除了上面的用法之外，INSERT还有另一种形式，就是可以将SELECT语句的结果插入表中，就是所谓的INSERT SELECT。例如将一张表的数据插入到另一张表可以这么做：

INSERT INTO Customers(cust_id, cust_name, cust_address, cust_city, cust_state, cust_zip, cust_country) SELECT cust_id, cust_name, cust_address, cust_city, cust_state, cust_zip, cust_country FROM CustNew;

同时，INSERT SELECT中所使用的是列的位置，因此两张表的列名并不需要匹配。而且，其中的SELECT语句可以包含WHERE子句来进行数据过滤。

小提示：

INSERT通常只插入一行。要插入多行，必须要执行多个INSERT语句。但是，INSERT SELECT是个例外，它可以用一条INSERT插入多行，不管SELECT语句返回多少行，都将被INSERT插入。

现在来看一下另一种技巧，从一个表复制到另一个表（不同的DBMS用法不同，只举例SQLite）：

CREATE TABLE CustCopy AS SELECT * FROM Customers;

在使用SELECT INTO时，要注意：

  • 任何SELECT选项和子句都可以使用，包括WHERE 和 GROUP BY；
  • 可利用连接从多个表插入数据了；
  • 不管从多少个表中检索数据，数据都只能插入到一个表中。

16.更新和删除数据

更新数据：UPDATE语句

更新或者修改表中的数据，可以使用UPDATE语句，有以下两种方式：

  • 更新表中特定行；
  • 更新表中所有行。

举个例子：

UPDATE Customers SET cust_email = 'xxx@xxx.com' WHERE cust_id = '10000000005';

有这个例子我们可以看出，基本的UPDATE由三个部分组成：

  • 要更新的表；
  • 列名和他们的新值；
  • 确定要更新的行的过滤条件。

同时，UPDATE可以更新多个列，用法如下：

UPDATE Customers SET cust_contact = 'Sam', cust_email = 'xxx@xxx.com' WHERE cust_id = '10000000005';

由此可知，在更新多个列时，只需要使用一条SET命令，每个“列=值”对之间用逗号分隔。

要删除某个列的值，可设置它为NULL（假如表定义允许NULL值）。如下进行：

UPDATE Customers SET cust_email = NULL WHERE cust_id = ‘1000000005’;

下面来看一下使用DELETE语句进行删除，他同样有两种使用方式：

  • 从表中删除特定的行；
  • 从表中删除所有行。

注意：不要省略WHERE子句。

使用方法如下：

DELETE FROM Customers WHERE cust_id = '1000000006';

17.创建和操纵表

利用CREATE TABLE创建表，必须给出下列信息：

  • 新表的名字，在关键字CREATE TABLE之后给出；
  • 表列的名字和定义，用逗号分隔；
  • 有的DBMS还要求指定表的位置。

使用例子如下：

CREATE TABLE Products ( 
prod_id CHAR(10)   NOT NULL,
vend_id CHAR(10)   NOT NULL,
prod_price  DECIMMAL(8,2) NOT NULL,
prod_desc   VARCHAR(1000)   NULL
)

更新表可以使用 ALTER TABLE语句

ALTER TABLE Vendors ADD vend_phone CHAR(20);

删除表则用DROP TABLE语句即可：

DROP TABLE CustCopy;
